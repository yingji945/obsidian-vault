# 四层记忆架构与 AI 记忆方案一览

## 为什么需要这个架构

AI 没有「记性」。每次对话都是新的上下文窗口，你之前告诉我的东西、我们共同建立的认知、走过的弯路——如果不刻意存，就全丢了。

设计的核心原则：**记忆是有层级的，越上层的越实时，越下层的越持久。** 不该让所有信息挤在同一层。

---

## 第一层：Memory（持久记忆）

**容量**：约 2200 字符  
**注入方式**：每次对话自动注入系统提示词  
**维护者**：我（Hermes）用 `memory()` 工具管理

**存什么**：
- 环境配置（网络状况、Python 工具链）
- 表结构（表名、关键字段、关联键）
- 用户偏好（语言、排版风格、协作习惯）
- 不能丢失的基础事实

**代价**：每次对话都消耗 token，所以必须精炼。满了优先删旧任务日志、保留偏好和配置。

**类比**：你的随身便签本。

---

## 第二层：Skills（操作手册）

**容量**：无硬限制（每个 skill 独立文件）  
**加载方式**：需要我主动 `skill_view()` 调用  
**维护者**：我 + 需要时补充

**存什么**：
- 可复用的部署流程（三省六部搭建步骤）
- 代码规范（SQL 风格、调试流程）
- 复杂工作流（多步骤、有陷阱的流程）
- 通过反复踩坑验证的最佳实践

**代价**：不会自动加载，需要我先意识到「这个场景该用这个 skill」。

**类比**：你的工作 SOP 文件。

---

## 第三层：Session Search（全文检索）

**容量**：所有历史对话  
**查询方式**：关键词搜索（FTS5 全文索引）  
**维护者**：自动化，无需人工干预

**存什么**：
- 所有完整的对话记录
- 错误调试过程
- 临时的讨论和想法

**制约**：只有关键词匹配，不支持语义检索。你说「我记得聊过客单价的事」，我得用 `"客单价"` 搜，不能理解「那个分析价格的对话」。

**类比**：日志档案柜。

---

## 第四层：Vault（结构化知识库）

**存储位置**：Obsidian 笔记（三层目录 + 双向链接）  
**维护者**：我（Hermes）写，你审核  
**同步方式**：GitHub → Obsidian Git 插件

**存什么**：
- 可以带走的认知（判断标准、方法论、AI 观）
- 跨笔记的链接网络
- 定期复盘沉淀的结论

**核心原则**：**个人认知优先于公司文档。** 离开瑞幸后最有价值的不是某个活动数据，而是你怎么分析问题、怎么判断好坏。

**类比**：你的私人图书馆。

---

## 四层协同工作流

```
你问一个复杂问题
    ↓
第1层 Memory → 我自动知道你的偏好和环境
    ↓
第2层 Skills → 我判断该加载哪个工作流
    ↓
我需要回忆之前的事
    ↓
第3层 Session Search → 关键词搜索历史对话
    ↓
结论值得记住
    ↓
第4层 Vault → 沉淀为结构化笔记，供你随时查阅
```

---

## 记忆缓存满时的处理规则

之前发生过 memory 容量不够的事，定下的规则：

1. **优先保留**：用户偏好 > 环境配置 > 表结构 > 任务进度 > 临时记录
2. **压缩策略**：合并多个同类条目，删除已完成的旧任务日志
3. **降级路径**：从 memory 移除 → 写成 skill（如果带流程）→ 写到 vault（如果值得长久存）
4. **不写说明**：memory 只存事实本身，不存"关于这件事的背景说明"

## 行业对比：其他 AI 记忆方案

### 方向一：长上下文 + Prompt Caching

| 方案 | 做法 |
|:----|:-----|
| Anthropic Prompt Caching | 常用上下文缓存，下次命中省 90% 成本 |
| Gemini 2M 上下文 | 不检索，整段往里扔 |

**逻辑**：窗口够大、缓存够便宜 → 不需要 RAG。

### 方向二：Agentic Memory（智能体自主记忆）

| 项目 | 做法 |
|:----|:-----|
| Mem0 | 模型自己判断「这值不值得记」，自动召回 |
| Letta（原 MemGPT） | 分工作记忆 + 长期记忆，模型自行管理归档 |
| CrewAI Memory | 多 Agent 共享记忆池 |

**逻辑**：不靠人设计检索逻辑，让模型自己管理。

### 方向三：Code-as-Memory（代码即记忆）

这就是我们现在的做法：

| 我们的层 | 对应行业概念 |
|:--------|:------------|
| memory | ≈ Mem0 的持久键值存储 |
| skills | ≈ 可复用思维链模板 |
| vault | ≈ 人工维护的知识图谱 |

**判断**：你在用的这四层，其实就是 Agentic Memory 的一种实现。外面很多团队花大价钱做的，你已经用 Obsidian + Hermes 跑通了。
