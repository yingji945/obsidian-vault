## ✅ 已完成（6项）

### 1️⃣ show_cost 开启 — `t_b2dddb5b`

**做了什么：** Hermes 配置中开启 `show_cost: true`

**为什么做：** 之前 HTTP 402 欠费后，想搞清楚钱花在哪了。开启后每条回复末尾显示当次对话消耗的 token 数和预估花费，帮你建立费用感知——哪些操作最费 token 一目了然。

**效果：** 每次对话末尾多一行花费显示，不影响使用。

---

### 2️⃣ git pull + push 重试 → 零 token 脚本 — `t_daa4aed5`

**做了什么：** 将两个 cron job（obsidian-vault git pull 每60分钟、git push 重试每60分钟）从 LLM 驱动模式改为 `no_agent=true` 纯脚本模式。

**为什么做：** 这两个任务只需要跑 `cd vault && git pull` 几个 shell 命令，之前每次却要调一次模型（日耗约 48 次模型调用），是最大的费钱元凶。HTTP 402 欠费后排查发现的问题。

**效果：** 立省每日 48 次模型调用，零 token 消耗。

---

### 3️⃣ max_turns 150→80 — `t_94250192`

**做了什么：** Hermes 配置中 `max_turns` 从 150 降到 80。

**为什么做：** 150 的上限过高，偶发的工具调用循环会导致大量 token 浪费。80 轮对正常对话完全够用（一般一次复杂任务 20-40 轮就完成），降低上限不影响正常使用。

**效果：** 防止跑飞浪费。

---

### 4️⃣ 飞书菜单配置 — `t_97f6738b`

**做了什么：** 在飞书开放平台 → 机器人 → 机器人菜单中配置了「发送消息」类型的菜单按钮：
- 📊 **查看用量** → 触发 `/usage` 命令，显示本轮对话 token 和花费
- 🔄 **切换模型** → 触发 `/model` 命令，查看当前模型，加模型名可切到 V4 Pro

**为什么做：** 原来只能手动输入斜杠命令，用户觉得不方便。配了菜单后点一下就能触发，不用打字。

**注意事项：** 选「发送消息」模式而非「自定义事件」模式，因为 Hermes 飞书端没有自定义事件处理器。发送消息模式原理就是点了自动帮你打一行字发出去，Hermes 正常解析成斜杠命令。
