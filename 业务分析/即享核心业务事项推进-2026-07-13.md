## 一、用户标签体系建设

### 覆盖范围扩容

| 原范围 | 新增范围 |
|:------|:---------|
| 即享首页、商品详情页访问、线下扫码、三方渠道触达 | + 专题页访问 + 秒杀页访问 |

### 标签分层规则

```
全量即享用户
    └── 是否产生即享商品下单行为
            ├── 是 → 即享老用户
            └── 否 → 即享新用户
```

### 约束规则

- **口径统一**：禁止随意创建自定义标签，所有标签由负责人统一维护
- **埋点规范**：新埋点统一加即享标识字段，底层自动识别归类
- **数据处理**：沿用 T+1 封版 + 当日实时统计规则，不额外开发
- **历史数据**：不回溯补全，仅新活动按规范执行
- **权限管控**：标签仅管理员可修改，普通账号只读；分组同理

### 奖励配置逻辑

```
用户领基础奖励 → 自主选择膨胀为其他权益
                    ↓
     膨胀参与率、转化率、消耗独立统计
```

### 权益类型

| 类型 | 统计规则 |
|:----|:---------|
| 现金红包 | 按打款成功数据 |
| 限制类优惠券 | 按用户实际核销 |
| 即享自有优惠券 | 同上，未核销不计入当期费用 |

## 四、数据看板建设 — 分三批

| 批次 | 优先级 | 内容 | 完成要求 |
|:----|:------|:-----|:---------|
| **P0** | 🔴 | 活动核心绝对值、转化率等基础指标 | **8月5日前必须完成** |
| **P1** | 🟡 | 关联门店、大盘的复合指标 | 逐步 |
| **P2** | 🟢 | 复购等复杂统计指标 | 临时SQL支撑，不等全量 |

### 核心统计口径

| 指标 | 口径 |
|:----|:-----|
| 曝光人数 | 页面访问的 OpenID 去重 |
| 参与人数 | 完成抽奖操作的 MemberID 去重 |
| 全渠道售卖量 | 按自然日，线下出库即计入（不等终端售卖） |
| 费用 | 仅核算真实发生：红包=打款成功，券=核销完成 |
| 维度提取 | 从现有统一活动记录表、订单表提取，不新增表结构 |
