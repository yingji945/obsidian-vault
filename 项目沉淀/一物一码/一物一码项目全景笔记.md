## 🎯 5 大核心领域

### 1. 活动方案设计

#### 奖品结构（最终版）

| 档位 | 奖品 | 膨胀选项 | 获得方式 |
|:----|:-----|:---------|:---------|
| 🥇 头奖 | 365张免费饮品券（内涵价值约25元/张） | ❌ 不可膨胀 | 加C/复购 |
| 🥈 二等奖 | 10张免费饮品券 | ❌ 不可膨胀 | 加C/复购 |
| 🥉 三等奖 | 1张饮品券（6.6元/9.9元） | ❌ 不可膨胀 | 加C/复购 |
| 💵 四等奖 | 1元现金红包 | ✅ 可膨胀为15元券/9.9元券（概率20%） | 加C/复购 |
| 💰 五等奖 | 0.5元现金红包 | ✅ 可膨胀（概率10%/35%/54%） | 复购 |
| 🎲 兜底 | 15元即享优惠券（60%） | ❌ | 直接领取 |

#### 加C（加私域）策略

| 奖品 | 加C策略 | 预估加C率 |
|:----|:--------|:---------|
| 365张券/10张券 | 强制加C | 100% |
| 1元红包 | 强制加C | 15~20% |
| 0.5元红包 | 前期强制加C测1个月 | 5% |
| 15元兜底券 | 加C领取 | 2% |

**目标：** 半年加C **88万**人，LTV目标**2000万**

#### AB测试设计

- **A组：** 加C领取奖品（又拆为已加C/未加C两组）
- **B组：** 复购第二瓶（48h内）领取奖品
- 分流比例：50%可调

### 2. 财务与税务

#### 税务规则

| 奖品类型 | 税务处理 |
|:---------|:---------|
| 💵 现金红包>5元 | 需代扣代缴**20%个税** → 取消99元/9.9元红包 |
| 💰 现金红包<4元 | **原则上不缴个税**（行业惯例） |
| 🎫 免费门店券 | 偶然所得，**20%个税** |
| 💳 9.9元购买券 | 差价（公允价值-购买价）×20%，差<4元可免 |
| 🏪 即享券（15元） | 销售折扣，**无额外税负** |
| 🏛️ 海南科技vs瑞幸中国 | 关键看是否同一主体+有无结算 |

#### 预算演变

```
初始方案（含大额红包）：~1,000万
V1方案（取消大额红包）：~956万
V2方案（概率化）：~870万
调整后（考虑税负+三等奖3%→0.5%）：~398万
最终（免费券比例调低）：~350万
```

#### 核算规则

| 奖项 | 核算方式 |
|:----|:---------|
| 现金红包 | 以用户**实际到账**时间 |
| 优惠券 | 领取时**预提**，使用时**确认成本**，过期**月末冲销** |
| 未核销权益 | **不计入**当期费用 |

### 3. 技术与系统

#### 系统架构

```
用户扫码
   │
   ▼
授权登录（微信/手机号）
   │
   ▼
抽奖引擎（100%中奖配置）
   │
   ├─ 现金红包 → 商户号打款至微信零钱
   ├─ 优惠券（门店咖啡券/即享券）
   │     └─ 加C领取（企微露娜）→ 券到账户
   └─ 膨胀玩法 → 二选一（原奖品/膨胀奖品）
         └─ 数据打点→Hive宽表→分析看板
```

#### 关键技术决策

| 决策 | 选择 |
|:----|:-----|
| 🔒 加密方案 | **方案二**（自动化加密，多花4万+10天） |
| 🏗️ 系统部署 | **私有化部署**（己方云端） |
| 🏷️ 商品编码 | **GS码**，瓶箱合一 |
| 🏪 结算方式 | 企业卡券系统（即享采购→活动码兑换） |
| 🗂️ 数据底座 | Hive + Oracle + Doris |
| 📊 看板策略 | 先搭**Hive宽表**+SQL，暂不做固化看板 |

#### 压测指标

- **目标：** 500 QPS（线上计算值80）
- **扩容策略：** 先不扩，利用现有容量探上限

### 4. 数据与统计

#### 统计筛选项（全链路漏斗）

```
曝光（OpenID去重）
   │
   ▼ 参与（完成抽奖操作，MemberID去重）
   │
   ▼ 中奖（100%中奖逻辑）
   │
   ▼ 选择奖品（原奖品/膨胀二选一）
   │
   ▼ 完成发放（加C/复购/直接领取）
```

#### 核心指标

| 指标 | 口径 |
|:----|:-----|
| 曝光人数 | 页面访问OpenID去重 |
| 参与人数 | 完成抽奖操作MemberID去重 |
| 加C率 | 1元15~20%，0.5元5%，大额100% |
| 复购率 | 第二瓶48h内扫码 |
| 膨胀率 | ~70%用户选择膨胀 |
| 核销率 | 咖啡券100%，即享券2% |

#### 数据表结构

- **主表：** `record_user_line`（用户中奖记录）
- **辅助表：** 活动奖品配置表、膨胀配置表、扫码流水表
- **关联键：** 奖品行ID → 活动奖品配置 / `expand_option_id` → 膨胀配置
- **存储：** Hive宽表 → 自定义SQL查询

### 5. 品牌与合规

#### 包装文案演变

```
初版: "有机会赢"（法务建议）
  → "100%中奖，最高赢365杯免费咖啡"（标签审核通过）
  → "免费限制咖啡"（替代"门店免费咖啡券"）
  → 最终：扫码赢门店红包
```

#### 合规要点

| 事项 | 要求 |
|:----|:-----|
| ✅ **去重可用** | 会员ID + 微信ID + 手机号 |
| ❌ **不得使用** | 第三方购买数据、门店碰撞数据、隐性设备ID |
| 📝 **规则展示** | **全量展示**（参考雀巢/农夫山泉），500字限制→放开 |
| 🔐 **隐私合规** | 扫码获取OpenID需签协议；仅传随机码给逸群 |
| ⚖️ **审核周期** | 上线前给李晶**4天复核** |
| 👩‍⚖️ **最高奖项** | **不能限制使用范围**（李晶强调） |
