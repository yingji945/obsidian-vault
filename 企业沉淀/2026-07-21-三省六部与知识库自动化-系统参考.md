# 三省六部 + 知识库自动化 — 系统参考手册

> 这不是一个实验项目，这是一套已经跑通的协作系统。
> 对应 skill: `multi-agent-roundtable`。

---

## 一、设计哲学

三省六部不是"我想跑一个 AutoGen 实验"，而是"我需要一个决策系统"。背后的几条原则：

1. **大脑+手脚** — Hermes 做编排大脑，不用自己做所有脏活
2. **圆桌而非并行** — Agent 互相能看到彼此、能反驳、能吵起来，不是各自写报告
3. **角色分界** — 数据大臣只管数据，运营大臣只管业务判断，法务只管合规
4. **每次成长** — 每轮圆桌的结论存下来，下次会前简报

---

## 二、三省六部角色定义

### 三省（决策层）

| 省 | 角色 | 职责 |
|---|---|---|
| 中书省 | 用户本人 | 最终决策，拍板 |
| 门下省 | 审核大臣（规划中） | 方案复核、风险把关 |
| 尚书省 | 调度大臣（规划中） | 任务协调、进度跟踪 |

### 六部（执行层）

| 部 | 大臣 | 专长 |
|---|---|---|
| 📊 户部 | 数据大臣 | SQL取数、漏斗分析、AB实验、埋点校验 |
| 🎯 礼部 | 运营大臣 | 活动策划、积分商城、用户触达、扫码活动 |
| 🧬 吏部 | 分层大臣 | 用户生命周期、留存分析、沉睡预警 |
| 🔬 工部 | 算法大臣（规划中） | 标签体系、预测模型 |
| ⚖️ 刑部 | 法务大臣 | 合规审查、备案、个税、隐私 |
| 🔍 兵部 | 情报大臣 | 竞品研究、行业动态 |

### 分阶段上线

- **Phase 1 ✅** 数据+运营+法务 3部先跑通
- **Phase 2 ✅** 加到5部（+分层+情报）
- **Phase 3 规划中** 全军出击+三省上线

## 四、已确认部署方案

### 运行命令

```bash
PYTHONPATH=~/.local/lib/python3.13/site-packages \
  python3 /opt/data/skills/multi-agent-roundtable/scripts/roundtable_live.py "你的议题"
```

### 已知坑（已全部修复）

| 问题 | 修复 |
|---|---|
| 非OpenAI模型缺少 `model_info` | 显式传 `model_info={"vision":False,...}` |
| Agent名称含中文报错 | 用英文名 + system_message 自我介绍 |
| SelectorGroupChat `{input}` 变量不存在 | 只能用 `{roles}/{participants}/{history}` |
| Selector 重复选同一人 | temperature 降到 0.5 + prompt 加"不要连续选同一个人" |
| 辩论页面无法访问 | 诊断流程：进程→容器→网关→公网IP逐层排查 |

## 五、知识库自动化（飞书→Obsidian Vault）

### 一句话流程

```
Jacob 23:06 沉淀飞书云文档
    → 23:08 群发5字段通知（📥/🏷️/📝/📄）
    → 刘小白 cron(82e221175075) 23:15 拉消息
    → 归类写 vault → git push
    → 老板 Obsidian 验证（23:15-24:00 否决窗口）
```

### 5字段通知格式

| 字段 | 内容 |
|---|---|
| 📥 今日知识沉淀 | 标题 |
| 🏷️ 分类 | 业务分析/个人成长/企业沉淀 |
| 📝 摘要 | 一句话说清内容 |
| 📄 链接 | 飞书文档链接 |
| 👤 发送人 | Jacob (cli_aa9fe0a4fa781cb6) |

### 方案 B 强规则

- 只有老板 @ 才响应，除此以外**绝对静默**
- 包括不主动发"保持安静"——连这条也不发
- **"沉默是一种主动行为"** — 3件不做事 = 主动执行方案B
